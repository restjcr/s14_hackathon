1. ¿Por qué utilizar un framework?

    - Porque nos ahorrará tiempo al utilizar código ya hecho. Porqué enmarca al equipo a trabajar con una guía, la del framework.

2. ¿Cuáles son los beneficios de usar un framework?

    - Un framework provee un patrón a seguir para el desarrollo. Además de adentrarnos a las buenas prácticas y facilidad de documentación.

3. ¿Cuáles son los problemas de no usar un framework?

    - Probablemente código sin una buena estructura, que afectará la escabilidad.
